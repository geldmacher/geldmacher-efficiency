# Efficiency Review

Use after implementation when the user wants to know whether the work avoided unnecessary Cursor cost.

1. Follow the `efficiency-review` skill.
2. Delegate the review to the readonly `efficiency-reviewer` agent when available.
3. Provide the original request, cost budget if any, actual changes, commands run, verification evidence, and known deviations.
4. Start with concrete waste, risk, or missed RTK opportunities.
5. Recommend cost-reduction adjustments only. 

Do not make efficiency review mandatory by default.
