# Efficiency Budget

Assess the task's likely cost drivers: broad or repeated context reads, noisy commands, unnecessary narration, oversized delegation, expensive checks, avoidable retries, risky external actions, and missing cheap evidence.

Return a compact recommendation covering the desired economy, the most valuable limits, proportional validation, actions that deserve explicit attention, and any measurement uncertainty. Treat these as task guidance, not as a replacement for user intent, project requirements, security controls, or Cursor's native approvals.

Use RTK project data when it is relevant and available. Distinguish same-session changes from cumulative statistics, calculations, estimates, and qualitative observations. Do not present cumulative RTK gain as savings from a single task or invent monetary cost without user-supplied rates.
